# Mobile Build (Capacitor)

The web app is wrapped with Capacitor to produce native Android and iOS builds. The web build is unaffected and continues to deploy to Vercel.

## Versions

- Capacitor 7 (pinned because Capacitor 8 requires Node ≥ 22; this workspace runs Node 20).
- Packages: `@capacitor/core`, `@capacitor/cli`, `@capacitor/android`, `@capacitor/ios` (all `^7`).

## Key Files

- `capacitor.config.json` — App id `com.hocvui.app`, name `Học Vui`, `webDir: "public"`.
- `public/index.html` — Capacitor entry point. Redirects to `home.html`.
- `public/api-config.js` — Runtime API base URL patch. Loaded via `<script src="api-config.js">` in every HTML file under `public/`. When running inside Capacitor (native shell or `capacitor:` / `file:` protocol) it monkey-patches `window.fetch` so that any `/api/...` call is rewritten to `https://<API_BASE_URL>/api/...`. Also rewrites root-relative WebSocket URLs to the same host. On a normal web origin it is a no-op.
- `scripts/inject-api-config.js` — Idempotent script that injects `<script src="api-config.js"></script>` into the `<head>` of every HTML file under `public/`. Re-run after adding new HTML pages.
- `android/` and `ios/` — Native projects generated by `npx cap add`. The `ios/` project requires macOS + Xcode + CocoaPods to build.

## NPM Scripts

- `npm run inject:api` — Inject `api-config.js` into any new HTML files.
- `npm run cap:sync` — Inject, then copy web assets into both native projects.
- `npm run cap:android` — Sync, then open the project in Android Studio.
- `npm run cap:ios` — Sync, then open the project in Xcode (macOS only).

## Configuration

Before producing a release build:

1. Edit `public/api-config.js` and set `API_BASE_URL` to the deployed API origin (e.g. `https://hoc-vui.vercel.app`, no trailing slash).
2. Run `npm run cap:sync` to propagate the change to `android/` and `ios/`.

## Online Duel (V4)

V4 uses Firebase Realtime Database directly from the client (see `public/v4/game.js`). It does **not** depend on the local `ws-server.js` or any `/api/` call, so it works in the native shell without extra hosting. The `ws-server.js` file is only used by the local dev server.

## Architecture Notes

- The native shell loads files from `capacitor://localhost` (iOS) or `https://localhost` (Android). Without `api-config.js`, `/api/...` calls would hit the device itself, so the patch is required for any backend-dependent feature.
- The patch is a no-op on the deployed Vercel site, so the web build continues to use same-origin API calls.
- Admin pages work in the native shell too, but Basic Auth prompts may behave oddly inside WebView. Prefer using admin from a desktop browser.
- For new HTML files added under `public/`, either run `npm run inject:api` once or paste `<script src="api-config.js"></script>` into the `<head>` (use a relative path matching the file's depth).

## Gotchas

- `npx cap doctor` can hang on Windows while it queries the npm registry. Skip it; it is not required for building.
- The `.gitignore` excludes generated native build artifacts (`android/.gradle`, `android/app/build`, `ios/App/Pods`, the copied `public/` mirrors inside `android/` and `ios/`, etc.) so only the source-controlled Capacitor scaffolding is committed.
- If the Android Studio build complains about missing SDK, install Android Studio and set `ANDROID_HOME`. iOS builds require macOS.
